# Core API Service

The `core-api` is the primary backend service for the LMS monorepo. It is a Go-based application providing a gRPC interface to various frontend clients and other systems.

## Architecture & Structure

This service follows a typical Go application layout with distinct layers for transport, business logic, and data access.

- **`cmd/server`**: Contains the entry point and server initialization logic (`run.go`). This is where dependencies are injected and the gRPC server is configured with its interceptors (logging, JWT auth, RBAC).
- **`internal/app`**: Wires the application together, holding references to all service handlers.
- **`internal/config`**: Configuration loading (environment variables) and default values.
- **`internal/db`**: Database connection initialization for Postgres (pgxpool) and Redis.
- **`internal/integrations`**: External service integrations, such as `r2` (Cloudflare R2 for media storage) and `sandbox` (external KYC API).
- **`internal/repository`**: Data access layer.
  - `schema/`: SQL table definitions.
  - `queries/`: SQL queries used by `sqlc`.
  - `generated/`: Go code generated by `sqlc` from the schemas and queries.
  - `migrations/`: Database migration scripts.
- **`internal/service`**: The core business logic, organized by domain (e.g., `admin`, `auth`, `dst`, `kyc`, `loan`, `media`, `onboarding`).
- **`internal/transport/grpc`**: The gRPC transport layer.
  - `generated/`: Go code generated from protobuf definitions.
  - `interceptors/`: gRPC interceptors for cross-cutting concerns like logging, authentication (JWT validation), and authorization (RBAC based on user roles).

## Dependency Injection & Routing

The service uses manual dependency injection in `cmd/server/run.go`. 
1. The config is loaded.
2. Infrastructure clients (Postgres, Redis, R2, Sandbox KYC) are initialized.
3. Repositories (sqlc queries) are instantiated.
4. Business services (`internal/service/*`) are created, injecting their required dependencies.
5. The `app.New()` function groups these handlers.
6. The gRPC server is registered with the respective service servers.

RBAC is enforced via `grpcinterceptors.RBACUnaryInterceptor`, which checks the method name against a predefined policy matrix mapping methods to allowed roles (e.g., `admin`, `manager`, `officer`, `borrower`, `dst`).

## Development

- **Run locally**: `go run .` (Ensure Postgres and Redis are running, or run them via Docker). Note that you need to set up appropriate environment variables.
- **Generate SQLC code**: Run `make sqlc` from the monorepo root.
